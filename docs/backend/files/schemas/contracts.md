# Source Code Guide: Backend Schemas (`schemas/`)

Schemas define the "Data Contracts" between the frontend and backend. They ensure that both sides agree on the structure of the data being exchanged.

---

## 1. `app/schemas/chat.py`

This file defines the inputs and outputs for the Chat and Agent endpoints.

### Key Classes
- **`ChatIn`**: The raw payload for a simple LLM chat.
    - `prompt`: The user's message.
    - `system_prompt`: Instructions for the AI's persona.
- **`ChatOut`**: The response from the LLM.
    - `content`: The text generated by the AI.
    - `raw_response`: The complete JSON from the LLM provider (useful for debugging).
- **`AgentPromptIn`**: A more complex input for the "Agent" endpoint.
    - Includes `enabled_tool_ids`: A list of tools the agent is allowed to use.
    - Includes `history`: Previous messages for maintaining context.
- **`AgentPromptOut`**: The comprehensive response from the agent.
    - Includes the AI's chat response.
    - Includes `tool_calls`: A list of specific tools the agent decided to run and their results.

---

## 2. `app/schemas/tooling.py`

This file defines the data structures for the Tool Registry and Execution.

### Key Classes
- **`ToolDefinitionOut`**: Describes a tool in the catalog.
    - `input_schema` / `output_schema`: JSON Schema objects that tell the frontend (and the AI) what arguments the tool needs.
    - `auth_type`: Whether the tool needs OAuth2 (like Gmail) or no auth (like Filesystem).
- **`ToolExecuteIn`**: The payload sent when a tool is manually executed.
    - `arguments`: A dictionary of values matching the tool's `input_schema`.
- **`ToolExecuteOut`**: The result of a tool execution.
    - `status`: "ok" or "error".
    - `trace_id`: A unique ID for tracking this specific execution in the logs.

---

## Why use Pydantic for Schemas?

1.  **Validation**: If a frontend sends a string where an integer is expected, Pydantic will automatically reject the request with a clear error message.
2.  **Documentation**: FastAPI uses these classes to generate the **Swagger UI** (at `/docs`). You can see the exact JSON structure required for every endpoint.
3.  **Type Safety**: In the backend code, we can access fields like `payload.prompt` instead of `payload["prompt"]`, which prevents many common bugs.
