# Weaver Learning Hub

Welcome to the comprehensive learning guide for **Weaver**, a tool-card-first multi-agent and automation platform. This documentation is designed to take you from a complete beginner to a deep understanding of the entire codebase.

## Documentation Structure

This hub is divided into three main sections:

### 1. [Getting Started](setup.md)
Start here to set up your local development environment for both the backend and frontend.

### 2. [Backend Deep Dive](backend/architecture.md)
Explore the Python/FastAPI backend.
- **[Architecture](backend/architecture.md)**: High-level design and tech stack.
- **[Module Overview](backend/modules.md)**: How the backend code is organized.
- [**Source Code Guide**](backend/files/index.md): Line-by-line explanations for every backend file.

### 3. [Frontend Deep Dive](frontend/architecture.md)
Explore the Flutter desktop application.
- [**Architecture**](frontend/architecture.md): UI structure and state management.
- [**Component Overview**](frontend/components.md): Key widgets and their roles.
- [**Source Code Guide**](frontend/files/index.md): Line-by-line explanations for every frontend file.

## How to use this Guide

This is not just a reference; it's a **tutorial**. We recommend the following flow:
1.  **Follow the [Setup Guide](setup.md)** to get the project running.
2.  **Read the [Backend Architecture](backend/architecture.md)** to understand how data flows.
3.  **Explore the [Backend Source Code](backend/files/main.md)** file by file.
4.  **Read the [Frontend Architecture](frontend/architecture.md)** to understand the UI logic.
5.  **Explore the [Frontend Source Code](frontend/files/main.md)** to see how the interface is built.

Every code snippet in the `files/` sections includes a line-by-line breakdown to ensure no detail is missed.

Happy Learning!
